Antes de iniciar a arquitetura do projeto, quero que você analise todos os documentos presentes na pasta docs/.

Objetivo:

Compreender profundamente o conteúdo que será utilizado pela IA do projeto Diabetes Guardian AI.

Os documentos incluem materiais da:

* ISPAD
* ADA
* Sociedade Brasileira de Diabetes
* Ministério da Saúde

Tarefas:

1. Leia todos os documentos disponíveis.

2. Identifique:

   * Título
   * Tema principal
   * Público-alvo
   * Capítulos
   * Assuntos abordados

3. Organize os documentos por categorias:

* Diagnóstico
* Aplicação de insulina
* Tipos de insulina
* Monitoramento glicêmico
* Hipoglicemia
* Hiperglicemia
* Cetoacidose diabética
* Alimentação
* Exercícios físicos
* Escola
* Saúde mental
* Tecnologias para diabetes
* Sensores de glicose
* Bombas de insulina
* Situações de emergência
* Educação para familiares

4. Identifique conteúdos duplicados entre documentos.

5. Identifique documentos mais relevantes para o RAG.

6. Crie uma matriz de relevância indicando:

* Documento
* Assunto
* Prioridade
* Utilidade para o assistente IA

7. Gere um resumo executivo contendo:

* O que a IA deve ser capaz de responder
* Quais temas possuem mais documentação
* Quais temas possuem pouca documentação
* Quais documentos serão a base principal do sistema

Arquivos de saída:

* document_analysis.md
* knowledge_map.md
* rag_sources_priority.md

Importante:

Não implemente nada ainda.

Primeiro compreenda completamente a base de conhecimento que será utilizada pelo sistema.
